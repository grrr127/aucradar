기능

“모든 정보를 다 보여주기”보다, 사용자가 실제로 필요로 하는 변화와 행동에 초점을 둡니다.

관심 물건(Watchlist)

물건을 저장하고, 한 화면에서 현재 상태를 확인할 수 있도록 구성합니다.

변경 감지

상태/기일/가격 등 주요 변동을 감지해 기록하고, 사용자에게 보여줍니다.

알림 설정

원하는 이벤트만 선택하여 불필요한 알림을 줄입니다.

필터/검색 (예정)

지역/법원/일정/금액대 등으로 빠르게 탐색할 수 있도록 확장합니다.

운영 로그 (운영자)

수집/갱신 상태를 확인하고 장애를 빠르게 파악할 수 있도록 합니다.

API 문서

DRF + Swagger로 API를 확인하고 연동할 수 있습니다. /api/docs/

참고: 기능 구현 순서는 “안정성 → 사용성 → 확장성”입니다. 우선 변동 감지 정확도를 올리고, 이후 화면/필터/알림 채널을 단계적으로 확장합니다.

데이터 흐름

수집
원문 기반으로 정보를 읽어옵니다.
정규화
필드/형식을 통일해 저장합니다.
변경 비교
이전 값과 비교해 변동을 식별합니다.
노출/알림
핵심 변동만 화면/알림으로 제공합니다.
